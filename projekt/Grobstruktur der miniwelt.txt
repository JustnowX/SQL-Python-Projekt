
Der "Proxmox Python SQL config extractor/injector" wurde von 2 IHK-Umschülern und einem Auszubildenden zum Fachinformatiker für Systemadministration ins Leben gerufen.

Zunächst stand der Spaß im Vordergrund, Dienste für die Klasse zu hosten. Dann kamen die ersten Dienste für Private und familiere Anwender hinzu.
Da der Verwaltungsaufwand der schier unendlichen Menge Virtueller Maschienen uns bereits nach Sekunden überforderte, entstand die Idee für den "Proxmox Python SQL config extractor/injector".

Die Virtuellen Maschienen werden zwar nach einem Schema erstellt, welches jedoch gewisse Freiheiten erlaubt.
So variieren z. B. die Menge der CPU Kerne, der RAM, das OS und vieles mehr. Leider konnte nicht immer der gewünschte Dienst mit gewünschter Hardware gestartet werden,
da die Hardware Ressourcen begrenzt sind. Ausserdem müssen manche Dinge, so zum Beispiel Ports, einmalig sein.



Der "Proxmox Python SQL config extractor/injector" benötig eine Datenbank zur Verwaltung folgender Punkte:
  -> Erfassung aller Virtuellen Maschienen und Ressources innerhalb von Proxmox Clustern!
  -> Zuteilung entsprechender Ressourcen


In weiteren Ausbaustufen ist geplant
  -> Reservierung von Hardware für künftige Projekte
  -> Visualisierung der Ressourcen mittels Graphen in Python
